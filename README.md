## Description

This is a simple getting started guide for connecting to a Beckhoff CX8290 running Beckhoff Realtime Linux.

The guide covers:

- Converting the CX8290 MAC address to an IPv6 link-local address
- Testing the connection using `ping`
- Connecting to the CX8290 using SSH
- Configuring Beckhoff package access
- Installing the TwinCAT 3 XAR package

## Prerequisites

You will need:

- A CX8290 connected to your network
- The MAC address of the CX8290
- A Windows PC connected to the same network
- Beckhoff package repository login credentials
- IPv6 enabled on your Windows network adapter

## Convert the MAC Address to an IPv6 Link-Local Address

To connect to the CX8290 for the first time, you need its IPv6 link-local address.

You can either use the official Beckhoff/Infosys procedure to manually convert the MAC address to an IPv6 link-local address, or you can use this helper tool:

[MAC to IPv6 Helper](https://benhar-dev.github.io/helper-mac-to-ipv6/)

Enter the MAC address of your CX8290 into the tool.

The tool will generate ready-to-use commands for:

- `ping`
- `ssh`

## Open Windows Terminal or Command Prompt

Open Windows Terminal or Command Prompt from the Start Menu.

You do not need to run it as Administrator.

## Test the Connection

First, test that the PC can reach the CX8290 using IPv6.

```bash
ping -6 <address>
```

Example:

```bash
ping -6 fe80::xxxx:xxxx:xxxx:xxxx
```

If the connection is successful, you should see ping replies listed in the terminal.

If you do not receive a reply:

- Check that the CX8290 is powered on
- Check that your Ethernet connection is correct
- Check that IPv6 is enabled on your Windows network adapter
- Confirm that the MAC address was entered correctly
- Use the exact command generated by the helper tool if it includes a Windows interface suffix

## Connect Using SSH

Once the ping test is successful, connect to the CX8290 using SSH.

```bash
ssh Administrator@<address>
```

Example:

```bash
ssh Administrator@fe80::xxxx:xxxx:xxxx:xxxx
```

You will be prompted for a password.

The default password is:

```text
1
```

Type the password and press Enter.

## Configure Beckhoff Package Access

> [!CAUTION]
> Passwords stored here are held in plain text. To reduce the risk of a credential leak, the authentication file can be saved temporarily and deleted after use (information for this can be found on Infosys). 

Once connected to the CX8290 over SSH, configure the Beckhoff package repository credentials.

Open the authentication file using `nano`:

```bash
sudo nano /etc/apt/auth.conf.d/bhf.conf
```

Paste the following content, replacing the example login and password with your own MyBeckhoff credentials:

```text
machine deb.beckhoff.com
login example@mail.com
password xyz123

machine deb-mirror.beckhoff.com
login example@mail.com
password xyz123
```

To save the file:

```text
Ctrl + O
```

Then press Enter.

To exit the editor:

```text
Ctrl + X
```

## Update Package Lists

Update the package lists:

```bash
sudo apt update
```

If the credentials are correct and the CX8290 has internet access, the package lists should update successfully.

## Install TwinCAT 3 XAR

Install the TwinCAT 3 XAR package:

```bash
sudo apt install tc31-xar-um
```

When prompted, type:

```text
y
```

Then press Enter to continue.

## Done!

You can now open your programming computer XAE and broadcast search to find your CX8290

## Quick Start Script (One-Liner)

For those of you who which to automate the configuration of your device, you can use .sh files.  
The command below will run the `./scripts/cx8290-setup.sh` script which will automate the last three steps of the above summary.

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/benhar-dev/cx8290-cheatsheet/main/scripts/cx8290-setup.sh)"
```

You can run the script multiple times without any issues.  I.e. if you need to change your email or password in the future.
